# Azure Agentic Aha (AAA) Deployment Framework

Azure Agentic Aha (AAA) is a *modular and flexible* [Infrastructure as Code (IaC)](https://learn.microsoft.com/devops/deliver/what-is-infrastructure-as-code) solution deployment framework for the [Microsoft Discovery Agentic AI](https://aka.ms/MicrosoftDiscovery) managed platform & [Azure High-Performance Computing (HPC)](https://learn.microsoft.com/azure/architecture/topics/high-performance-computing) infrastructure.

The following solution design principles and features are implemented throughout the AAA deployment framework.
* Defense-in-depth layered security with core service integration across [Managed Identity](https://learn.microsoft.com/entra/identity/managed-identities-azure-resources/overview), [Key Vault](https://learn.microsoft.com/azure/key-vault/general/overview), [Private Link](https://learn.microsoft.com/azure/private-link/private-link-overview),</br>[Private Endpoints](https://learn.microsoft.com/azure/private-link/private-endpoint-overview), [Network Security Groups](https://learn.microsoft.com/azure/virtual-network/network-security-groups-overview), [Network Security Perimeter](https://learn.microsoft.com/azure/private-link/network-security-perimeter-concepts), [NAT Gateway](https://learn.microsoft.com/azure/nat-gateway/nat-overview), [Bastion](https://learn.microsoft.com/azure/bastion/bastion-overview), [Firewall](https://learn.microsoft.com/azure/firewall/overview), etc
* Integration of [Monitor](https://learn.microsoft.com/azure/azure-monitor/fundamentals/overview) data collection and log analytics workspace with [Managed Grafana](https://learn.microsoft.com/azure/managed-grafana/overview) data visualization
* Any 3<sup>rd</sup>-party or custom application software is supported in a custom [Compute Gallery](https://learn.microsoft.com/azure/virtual-machines/azure-compute-gallery) virtual machine image
* Multi-Region and [Extended Zone](https://learn.microsoft.com/azure/extended-zones/overview) deployments are supported via a peered [Virtual Network](https://learn.microsoft.com/azure/virtual-network/virtual-networks-overview) per region or zone
* Clean separation of module *configuration* files (**config.auto.tfvars**) and *implementation* files (**\*.tf**) via [Terraform](https://learn.microsoft.com/azure/developer/terraform/overview)

## Local Installation Process

The following local installation process is required for module deployment orchestration.

1. Make sure the [Azure CLI](https://learn.microsoft.com/cli/azure/install-azure-cli) is installed locally and accessible in your PATH environment variable.
2. Make sure the [Terraform CLI](https://developer.hashicorp.com/terraform/downloads) is installed locally and accessible in your PATH environment variable.
3. Clone this GitHub repository to your local workstation for module configuration and deployment.

## Module Configuration & Deployment

The following table describes the module configuration and deployment process using Terraform commands on your local workstation.

| **Module Name** | **Module Configuration** |
| - | - |
| All Modules | 1. Review and edit the default `config.auto.tfvars` settings for your target Azure deployment.<br/>2. For module `0.Foundation` only, run `terraform init` in your local directory command shell (append `-upgrade` as needed). For each other module, run `terraform init -backend-config ../0.Foundation/backend.config` (append `-upgrade` as needed).<br/>3. Run [terraform plan](https://developer.hashicorp.com/terraform/cli/run#terraform-plan) in your local directory command shell and review the displayed deployment plan (i.e., adds, changes & destroys).<br/>4. Run [terraform apply](https://developer.hashicorp.com/terraform/cli/run#terraform-apply) in your local directory command shell (append `-destroy` as needed to delete all module resources in Azure).<br/>**5. *Before confirming `terraform apply`*, carefully review the displayed deployment plan (i.e., adds, changes & destroys).**
| [0&#160;Foundation](https://github.com/Azure/AgenticAha/tree/main/0.Foundation) | 1. Ensure the current user has the required [Azure Key Vault Administrator](https://learn.microsoft.com/azure/role-based-access-control/built-in-roles/security#key-vault-administrator) RBAC role assignment.<br/>2. Review and edit the `backend.config` settings for Azure Blob Storage Terraform state files.<br/>
| [2 Image](https://github.com/Azure/AgenticAha/tree/main/2.Image) | 1. Ensure you have sufficient **Standard** compute cores quota approved on your Azure subscription for each configured VM type / size in your target region.<br/>2. Use the Azure portal or [Image Builder CLI](https://learn.microsoft.com/cli/azure/image/builder#az-image-builder-run) to start an **Image Template** build run.
| [6 HPC](https://github.com/Azure/AgenticAha/tree/main/6.HPC) | 1. Ensure you have sufficient **Standard** and/or [Spot](https://learn.microsoft.com/azure/virtual-machines/spot-vms) compute cores quota approved on your Azure subscription for each configured VM type & size in your target region.<br/>2. Ensure **image** references in `config.auto.tfvars` point to your custom [Compute Gallery](https://learn.microsoft.com/azure/virtual-machines/azure-compute-gallery) image.
| [7 VDI](https://github.com/Azure/AgenticAha/tree/main/7.VDI) | 1. Ensure you have sufficient **Standard** compute cores quota approved on your Azure subscription for each configured VM type & size in your target region.<br/>2. Ensure **image** references in `config.auto.tfvars` point to your custom [Compute Gallery](https://learn.microsoft.com/azure/virtual-machines/azure-compute-gallery) image.
